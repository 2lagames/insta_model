# Plan: Instagram Content Factory For A Female Model

Дата обновления: 2026-06-24

## 1. Цель

Построить контент-завод для ведения Instagram-аккаунта девушки-модели: находить тренды, анализировать их пригодность, превращать тренды в конкретные фото/видео-идеи, генерировать или готовить материалы через API, отдавать готовый пакет для публикации/выкладки.

На первом этапе не строим Mac-приложение. Сначала фиксируем процесс, источники данных, критерии анализа, ограничения API и MVP-дорожную карту. Этот файл ведем как главный рабочий план.

## 2. Главный принцип продукта

Система не должна просто "генерировать красивую картинку". Она должна отвечать на цепочку:

1. Почему этот тренд стоит брать?
2. Как именно он работает в Instagram?
3. Подходит ли он образу модели и целевой аудитории?
4. Как адаптировать тренд так, чтобы контент выглядел нативно, а не как копия?
5. Какой конкретный материал нужно сделать: фото, Reels, карусель, сторис?
6. Какие промпты, референсы, подписи и хештеги нужны?
7. Готов ли материал к публикации после человеческой проверки?

## 3. Важные ограничения Instagram/API

Официальный путь - Meta/Instagram Graph API и Instagram Platform APIs. Они полезны, но не дают полного "радара трендов" как в Explore/Reels внутри приложения.

Проверенные направления:

- Instagram Platform позволяет работать с профессиональными аккаунтами: Business и Creator. Источник: https://developers.facebook.com/docs/instagram-platform/overview/
- Content Publishing API позволяет публиковать изображения, видео, Reels и карусели через официальный API. Источник: https://developers.facebook.com/docs/instagram-platform/content-publishing/
- Hashtag Search API позволяет искать хештег и получать `top_media` / `recent_media` по нему. Источник: https://developers.facebook.com/docs/instagram-platform/instagram-api-with-facebook-login/hashtag-search/
- Instagram Media Insights дает метрики по собственным публикациям аккаунта. Источник: https://developers.facebook.com/docs/instagram-platform/reference/instagram-media/insights/
- Graph API имеет лимиты запросов и бизнес-лимиты использования. Источник: https://developers.facebook.com/docs/graph-api/overview/rate-limiting/

Практический вывод:

- Официальный API подходит для публикации, аналитики собственного аккаунта, анализа хештегов и части медиа.
- Официальный API не заменяет полноценный поиск трендов в Reels/Explore/аудио.
- Для тренд-радарa понадобится гибрид: официальные API + ручные/полуавтоматические подборки + внешние тренд-источники + собственная база результатов.
- Серый scraping Instagram надо рассматривать как риск: блокировки, нарушение ToS, нестабильность, юридические проблемы.

## 4. Источники поиска трендов

### 4.1. Instagram-источники

1. Хештеги через Meta API:
   - список базовых хештегов по нишам;
   - `top_media` для поиска сильных форматов;
   - `recent_media` для оценки свежести;
   - динамика вовлечения, если доступна.

2. Собственный аккаунт:
   - какие посты набирают охваты, сохранения, репосты, просмотры;
   - какие темы и визуальные стили лучше заходят;
   - какие форматы дают подписки, а не только лайки.

3. Ручной watchlist:
   - аккаунты моделей в похожей нише;
   - fashion/beauty/fitness/lifestyle creators;
   - региональные аккаунты, если аудитория локальная;
   - подборки Reels, аудио, монтажных приемов.

4. Ручная разметка трендов:
   - ссылка на пример;
   - тип тренда;
   - визуальная формула;
   - звук/музыка;
   - хук;
   - что повторяется в лучших примерах;
   - как адаптировать под нашу модель.

### 4.2. Внешние источники

1. TikTok Creative Center / TikTok тренды - часто тренды приходят в Instagram Reels с задержкой.
2. Google Trends - широкие темы, сезоны, события, бренды, запросы.
3. Pinterest Trends - визуальные направления, fashion/beauty/lifestyle.
4. YouTube Shorts / YouTube trending - форматы короткого видео.
5. Reddit / X / Telegram / профильные медиа - ранние мемы и темы.
6. Платные или полуофициальные social listening tools: Modash, HypeAuditor, Metricool, Later, Sprout Social, Phyllo, Ayrshare, Zernio и аналоги. Их надо отдельно сравнить по цене, API, ToS и доступу к Instagram-данным.

## 5. Типы трендов

1. Визуальный тренд:
   - позы;
   - свет;
   - кадрирование;
   - цветокор;
   - локации;
   - wardrobe/style.

2. Reels-формат:
   - монтажный прием;
   - before/after;
   - переходы;
   - POV;
   - lip sync;
   - loop-видео;
   - "3 кадра за 5 секунд";
   - slow reveal.

3. Аудио-тренд:
   - популярный звук;
   - музыка;
   - реплика;
   - синхронизация движения с битом.

4. Текстовый/мемный тренд:
   - подписи на экране;
   - relatable-ситуации;
   - self-irony;
   - контраст ожидание/реальность.

5. Сезонный/событийный тренд:
   - праздники;
   - погода;
   - fashion seasons;
   - локальные события;
   - инфоповоды.

## 6. Как анализировать тренд

Каждый найденный тренд превращаем в Trend Card.

### 6.1. Поля Trend Card

- `trend_id`
- `date_found`
- `source`
- `source_links`
- `trend_type`
- `short_description`
- `example_posts`
- `audio_or_sound`
- `visual_formula`
- `hook_formula`
- `caption_patterns`
- `hashtag_cluster`
- `target_audience`
- `fit_for_model`
- `production_complexity`
- `risk_level`
- `freshness_score`
- `virality_score`
- `brand_fit_score`
- `content_ideas`
- `recommended_format`
- `status`

### 6.2. Оценка тренда

Оцениваем по шкале 1-5:

- Freshness: насколько тренд свежий.
- Velocity: как быстро растет количество примеров/упоминаний.
- Saturation: не перегрет ли тренд.
- Fit: подходит ли образу модели.
- Production cost: насколько сложно снять/сгенерировать.
- Repeatability: можно ли сделать серию, а не один пост.
- Risk: есть ли риск бана, плохого восприятия, нарушения авторских прав.
- Conversion potential: может ли тренд приводить подписчиков, заявки, продажи, узнаваемость.

Итоговая формула для MVP:

```text
trend_score = freshness * 0.20
            + velocity * 0.20
            + fit * 0.25
            + repeatability * 0.15
            + conversion_potential * 0.10
            - saturation * 0.05
            - risk * 0.05
```

Формулу будем менять после первых тестов.

## 7. Контент-пайплайн

### 7.1. Общий поток

1. Trend Discovery:
   - собрать тренды из Instagram, внешних источников и ручного watchlist;
   - сохранить кандидатов в базу.

2. Trend Analysis:
   - заполнить Trend Card;
   - оценить по формуле;
   - выбрать тренды со статусом `approved_for_ideation`.

3. Content Ideation:
   - на каждый тренд создать 3-10 адаптаций;
   - определить формат: фото, карусель, Reels, сторис;
   - прописать хук, сценарий, визуальный стиль, подпись.

4. Creative Brief:
   - собрать референсы;
   - выбрать образ модели;
   - выбрать локацию/фон;
   - выбрать wardrobe;
   - прописать промпты для генерации;
   - прописать негативные ограничения.

5. Generation / Production:
   - фото через image generation API;
   - видео через video generation API;
   - голос/озвучка через TTS, если нужно;
   - монтаж/субтитры через отдельный render pipeline.

6. QA:
   - проверить лицо/руки/анатомию;
   - проверить соответствие образу;
   - проверить отсутствие артефактов;
   - проверить текст на экране;
   - проверить права на музыку/референсы;
   - проверить формат Instagram.

7. Delivery:
   - финальное фото/video;
   - обложка;
   - подпись;
   - хештеги;
   - alt text;
   - рекомендации по времени публикации;
   - пометка "можно публиковать" или "нужна доработка".

## 8. Роль AI/API

### 8.1. Аналитика

AI помогает:

- кластеризовать тренды;
- извлекать повторяющиеся паттерны из описаний и примеров;
- сравнивать тренд с образом модели;
- генерировать идеи адаптаций;
- писать сценарии Reels;
- делать промпты для изображений/видео;
- писать подписи и варианты хуков.

### 8.2. Генерация

Возможные API-блоки:

- image generation: создание фото, вариаций, обложек, backgrounds;
- image editing: ретушь, замена фона, адаптация одежды/стиля;
- video generation: короткие Reels, motion shots, transitions;
- TTS/audio: озвучка, voiceover;
- captioning: субтитры;
- render/export: сборка финального MP4.

Конкретные провайдеры выбираем после тестов по качеству, цене, скорости, API и правам использования.

## 9. Безопасность, права и этика

Перед генерацией контента с моделью нужно зафиксировать:

- модель совершеннолетняя;
- есть согласие на использование внешности/референсов;
- есть разрешенные референс-фото;
- запрещены несанкционированные lookalike-образы других людей;
- запрещены материалы, которые нарушают правила Instagram;
- музыка и чужие видео используются только в рамках разрешенного использования;
- если аккаунт коммерческий, учитываем рекламные disclosure-требования.

## 10. MVP без Mac-приложения

Первый MVP должен быть максимально простым:

```text
trend sources -> trend_cards/*.md or .json -> AI analysis -> content_briefs/*.md -> generated_assets/ -> delivery_packages/
```

### 10.1. Файловая структура MVP

```text
/plan.md
/data/
  /trend_sources/
  /trend_cards/
  /account_insights/
/briefs/
  /content_briefs/
/assets/
  /model_references/
  /style_references/
  /generated/
/deliveries/
  /ready_to_publish/
  /needs_review/
/scripts/
  /collect_trends/
  /score_trends/
  /generate_briefs/
```

### 10.2. Что должно получиться в первом MVP

1. Можно добавить тренд вручную или через простой collector.
2. Система создает Trend Card.
3. Система оценивает тренд.
4. Система предлагает 3-5 идей под модель.
5. Система создает один Content Brief.
6. Система генерирует промпты для фото/видео.
7. Человек выбирает вариант.
8. Система складывает готовый пакет в `deliveries/ready_to_publish/`.

## 11. Будущее Mac-приложение

Mac-приложение лучше строить после MVP, когда станет ясно, какие действия реально повторяются каждый день.

Решение от 2026-06-24 для первого интерактивного среза:

- идем через Tauri + React;
- сначала делаем браузерный/React-прототип с hot reload и локальным Node API;
- первый экран - Studio Workspace;
- первая функция - импорт Instagram URL через `yt-dlp`;
- `yt-dlp` запускаем с `--ignore-config --cookies-from-browser chrome`, чтобы не наследовать сторонний пользовательский конфиг и пробовать авторизованные Chrome cookies;
- для видео извлекаем первый кадр через `ffmpeg`;
- снизу показываем галерею импортированных материалов; если пост является каруселью, каждое фото/видео показывается отдельной карточкой;
- справа оставляем место под будущую генерацию фото/видео и анализ трендов.
- текущая машина уже имеет Node/npm, `yt-dlp` и `ffmpeg`, но не имеет Rust toolchain; полноценный Tauri wrapper потребует установки Rust перед запуском `tauri dev`.

Вероятная архитектура:

- Tauri Mac app;
- React + Vite frontend;
- на первом runnable MVP: Node/Express API для локальных операций;
- позже: Rust command layer Tauri заменит Node API без изменения основной логики UI;
- локальное хранение файлов и JSON-индекс на первом MVP;
- SQLite позже, когда появятся тренды, briefs, generation jobs и история публикаций;
- Keychain позже для API-ключей;
- локальная media library;
- экран Import Studio;
- экран Trend Radar;
- экран Trend Card;
- экран Content Brief;
- экран Generation Queue;
- экран Review/Approval;
- экспорт в папку или публикация через Instagram API.

Важно: автопубликацию лучше включать только после стабильной ручной проверки. На раннем этапе безопаснее делать "готовый пакет для публикации".

## 12. Этапы разработки

### Этап 0. Уточнение стратегии

- выбрать нишу модели;
- выбрать язык и географию аудитории;
- определить границы контента;
- выбрать: реальная модель, AI-модель или гибрид;
- определить частоту публикаций;
- определить KPI.

### Этап 1. Ручной тренд-радар

- создать шаблон Trend Card;
- создать список хештегов;
- создать список конкурентов/watchlist;
- вручную собрать первые 30 трендов;
- проверить, какие поля реально нужны.

### Этап 2. Полуавтоматический сбор

- подключить Instagram Hashtag Search API;
- выгружать top/recent media по выбранным хештегам;
- подключить Google Trends/Pinterest/TikTok как внешние источники, если API/экспорт доступен;
- сохранять результаты в JSON/SQLite.

### Этап 3. AI-анализ

- автоматически заполнять Trend Card;
- считать score;
- кластеризовать похожие тренды;
- выделять лучшие варианты для модели;
- генерировать первые Content Briefs.

### Этап 4. Генерация контента

- выбрать 1-2 image/video API;
- собрать библиотеку референсов модели;
- сделать prompt templates;
- генерировать тестовые фото;
- генерировать короткие видео;
- добавить QA checklist.

### Этап 5. Delivery-пакет

- сохранять готовые материалы по папкам;
- добавлять caption, hashtags, alt text;
- добавлять preview image;
- добавлять publish checklist;
- экспортировать единый пакет.

### Этап 6. Mac-приложение

- перенести повторяемые операции в UI;
- добавить очередь генерации;
- добавить review workflow;
- добавить настройки API;
- добавить локальное хранилище;
- добавить интеграцию с Instagram publishing API, если нужна автопубликация.

## 13. KPI

Для аккаунта:

- reach;
- Reels plays;
- saves;
- shares;
- profile visits;
- follows;
- engagement rate;
- retention на видео;
- replies/DMs;
- conversions, если есть коммерческая цель.

Для фабрики:

- количество найденных трендов в день;
- процент трендов, дошедших до brief;
- процент briefs, дошедших до публикации;
- время от тренда до готового материала;
- стоимость одного готового материала;
- доля материалов, прошедших QA с первого раза.
