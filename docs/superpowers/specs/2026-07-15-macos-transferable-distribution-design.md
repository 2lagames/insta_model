# Передаваемый установочный пакет для macOS — дизайн

## Цель

Подготовить безопасный комплект Instagram Import Studio, который можно передать
другому человеку одним ZIP-архивом. Получателю не нужен доступ к GitHub: он
распаковывает архив, запускает `Установить.command`, а после первой установки
запускает приложение через `Запустить.command`.

Пакет не получает автоматические обновления. Для передачи новой версии владелец
создаёт и отправляет новый архив.

## Целевая среда

- macOS на Apple Silicon или Intel.
- На первой установке требуется интернет для Homebrew, Node.js LTS и npm-пакетов.
- Если Homebrew отсутствует, используется его официальный установщик; macOS может
  запросить пароль администратора.
- Приложение запускается локально и открывает `http://localhost:5173`.

## Состав и изоляция

В пакет попадает только минимальный snapshot файлов, необходимых для запуска
приложения. Это не Git checkout: каталог `.git`, ветки, теги и история коммитов
отсутствуют.

Сборщик явно запрещает и проверяет наличие следующих путей и данных:

- `.git`, `.github`, `.githooks`;
- `data`, `input`, `output`, `node_modules`, `dist`;
- `.env`, `*.local.json`, ключевые и сертификатные файлы;
- локальные API-ключи из `data/connections.local.json`;
- установщики и обновляторы, обращающиеся к приватному GitHub.

Исходный snapshot приложения присутствует в `app/`, поскольку выбран лёгкий
кросс-архитектурный вариант без Electron и встроенного Node.js. Приватный GitHub
и его история получателю не раскрываются.

Сборщик создаёт distribution-вариант `package.json`: зависимости и команды
запуска остаются зафиксированными, но `dev` не вызывает `setup:git-hooks`, а
служебные команды `setup:git-hooks`, `check:secrets`, `test` и `test:watch`
удаляются. `package-lock.json` сохраняется без изменения, потому что набор и
версии зависимостей не меняются. Тестовые `*.test.ts` и каталог `scripts` в
runtime-snapshot не включаются.

## Установка

`Установить.command` выполняет следующие действия:

1. Переходит в каталог пакета независимо от места запуска.
2. Проверяет, что выполняется на macOS.
3. Устанавливает Homebrew официальным способом, если он отсутствует.
4. Активирует Homebrew из `/opt/homebrew` или `/usr/local`.
5. Устанавливает Node.js, если `node` или `npm` отсутствуют.
6. Выполняет `npm ci` внутри `app/`.
7. Запускает локальное приложение в foreground через `npm run dev`.

Git не устанавливается и не используется. Скрипт не содержит URL приватного
репозитория и не выполняет `git clone`, `git fetch` или `git pull`.

Ошибки выводятся понятным текстом, окно терминала остаётся открытым до нажатия
Enter. Повторный запуск установщика безопасен: `npm ci` восстанавливает строго
зафиксированные зависимости и не удаляет локальные runtime-каталоги приложения.

## Последующий запуск

`Запустить.command` проверяет Node.js/npm и наличие `node_modules`. Если первая
установка не завершена, пользователь получает инструкцию запустить
`Установить.command`. При корректной установке скрипт запускает приложение в
foreground; закрытие терминала останавливает локальный сервер.

Автоматических обновлений, фоновых служб и сетевых запросов к GitHub нет.

## Локальные данные пользователя

После первого запуска приложение создаёт `app/data`, `app/input` и `app/output`
только на компьютере получателя. API-ключи вводятся самим пользователем на листе
`Подключения` и сохраняются в `app/data/connections.local.json`. Пакет сборки не
содержит этот файл и не переносит данные владельца приложения.

Обычная работа приложения по-прежнему обращается к выбранным пользователем
интеграциям ScrapeCreators, Ollama и RunningHub. Это отдельно от установки и не
связано с GitHub.

## Документация получателя

`README.txt` содержит короткую последовательность:

1. Распаковать ZIP в обычную пользовательскую папку.
2. Запустить `Установить.command` правой кнопкой через **Открыть**, если macOS
   показывает предупреждение для загруженного скрипта.
3. Дождаться открытия браузера и ввести собственные API-ключи.
4. В дальнейшем использовать `Запустить.command`.
5. Не перемещать отдельные файлы из распакованной папки и не ожидать
   автоматических обновлений.

## Сборка и проверка

В основном приватном репозитории создаётся воспроизводимый сборщик
`scripts/build-macos-distribution.mjs` и контрактные тесты. Сборщик сначала
создаёт новый временный каталог, копирует allowlist файлов, генерирует два
`.command`-файла и `README.txt`, затем запускает проверки до публикации результата.

Обязательные проверки:

- `npm test` и `npm run build` исходного проекта;
- контрактные тесты состава, URL и поведения установочных скриптов;
- `bash -n` для обоих `.command`-файлов;
- отсутствие запрещённых путей в развёрнутой папке и ZIP;
- сканирование всех текстовых файлов на распространённые сигнатуры секретов;
- сравнение содержимого пакета с фактическими локальными API-ключами без вывода
  значений;
- распаковка ZIP в новый временный каталог;
- `npm ci`, запуск приложения, успешный ответ `/api/health`, остановка только
  созданного тестового процесса.

Сборщик заменяет существующий результат только после успешного создания и
проверки нового временного пакета. Локальные runtime-данные основного проекта
никогда не копируются в каталог дистрибутива.
