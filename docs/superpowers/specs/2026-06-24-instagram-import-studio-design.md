# Instagram Import Studio Design

Дата: 2026-06-24

## 1. Цель первого MVP

Собрать первый интерактивный срез будущего Mac-приложения для контент-завода Instagram: пользователь вставляет ссылку на Instagram-пост или Reels, приложение скачивает медиа через `yt-dlp`, автоматически определяет фото это или видео, для видео извлекает первый кадр через `ffmpeg`, показывает результат в интерфейсе и сохраняет материалы для дальнейшей работы.

Этот MVP не делает генерацию изображений или видео. Он оставляет для нее отдельную зону интерфейса, чтобы будущие функции добавлялись без перестройки основного экрана.

## 2. Подтвержденные продуктовые решения

- Формат разработки: браузерный интерактивный прототип с последующей упаковкой в Mac-приложение.
- Технический путь: Tauri + React.
- Первый экран: Studio Workspace.
- Основной workflow: одна Instagram-ссылка -> импорт -> preview -> нижняя галерея материалов.
- Первый уровень хранения: локальные файлы и JSON-индекс.
- Генерация изображений/видео не входит в первый MVP, но под нее резервируется правая рабочая зона.

## 3. Пользовательский сценарий

1. Пользователь открывает приложение.
2. Вставляет Instagram URL в верхнее поле.
3. Нажимает `Import`.
4. Приложение показывает статус: подготовка, загрузка, анализ, извлечение кадра, готово или ошибка.
5. Backend-команда вызывает `yt-dlp`.
6. `yt-dlp` сохраняет медиа и metadata.
7. Приложение определяет тип результата:
   - `image`: показывает фото;
   - `video`: показывает видео и извлеченный первый кадр;
   - `carousel`: сохраняет все полученные элементы, если `yt-dlp` их вернул.
8. Для видео backend вызывает `ffmpeg` и создает first frame image.
9. Результат появляется в нижней галерее.
10. Клик по элементу галереи открывает его в центральном preview.
11. Пользователь может открыть папку с файлами.

## 4. Интерфейс

### 4.1. Верхняя панель

Содержит:

- поле ввода Instagram URL;
- кнопку `Import`;
- кнопку `Open Folder`;
- короткий статус текущей операции.

### 4.2. Центральная зона preview

Показывает выбранный материал:

- фото;
- видео-плеер;
- first frame для видео;
- metadata выбранного импорта: source URL, local path, media type, дата импорта.

### 4.3. Правая зона будущей генерации

В первом MVP зона отключена или работает как placeholder. Она визуально показывает будущие направления:

- image generation;
- video generation;
- trend analysis;
- caption/hashtag generation.

Кнопки будущих функций должны быть disabled, чтобы пользователь понимал, что это место зарезервировано, но не ожидал работающей генерации.

### 4.4. Нижняя галерея

Показывает все полученные материалы:

- thumbnail фото;
- thumbnail видео;
- first frame;
- индикатор типа;
- статус импорта.

Галерея должна позволять переключаться между материалами одним кликом.

## 5. Архитектура

### 5.1. Frontend

React + Vite:

- управляет состоянием формы;
- вызывает Tauri commands;
- показывает прогресс;
- отображает preview;
- отображает галерею;
- читает JSON-индекс импортов через backend command.

### 5.2. Tauri shell

Tauri отвечает за:

- запуск локальных команд;
- доступ к файловой системе;
- открытие папки с материалами;
- безопасный мост между UI и backend-командами.

### 5.3. Backend command layer

Rust-команды Tauri:

- `import_instagram_url(url: String) -> ImportResult`
- `list_imports() -> Vec<ImportItem>`
- `open_imports_folder()`

Команда `import_instagram_url`:

1. валидирует URL;
2. создает папку импорта;
3. запускает `yt-dlp`;
4. читает metadata JSON;
5. классифицирует медиа;
6. запускает `ffmpeg` для видео;
7. обновляет JSON-индекс;
8. возвращает результат в UI.

## 6. Локальное хранение

Первый MVP хранит данные в папке приложения или проекта:

```text
data/
  imports/
    index.json
    <import_id>/
      source.json
      media.jpg
      media.mp4
      thumbnail.jpg
      first_frame.jpg
      yt_dlp.info.json
```

`index.json` содержит список импортов:

```json
{
  "items": [
    {
      "id": "20260624-153000-abc123",
      "sourceUrl": "https://www.instagram.com/reel/example/",
      "mediaType": "video",
      "status": "ready",
      "createdAt": "2026-06-24T15:30:00Z",
      "files": {
        "video": "data/imports/20260624-153000-abc123/media.mp4",
        "firstFrame": "data/imports/20260624-153000-abc123/first_frame.jpg",
        "metadata": "data/imports/20260624-153000-abc123/yt_dlp.info.json"
      }
    }
  ]
}
```

## 7. Команды и зависимости

Для MVP требуются:

- `yt-dlp`;
- `ffmpeg`;
- Node.js;
- Rust toolchain;
- Tauri CLI.

На первом этапе допустимо требовать установленные в системе `yt-dlp` и `ffmpeg`. Позже можно решить, нужно ли бандлить их внутрь приложения.

Проверки запуска:

- если `yt-dlp` не найден, UI показывает инструкцию установки;
- если `ffmpeg` не найден, видео скачивается, но first frame помечается как failed;
- если Instagram требует cookies/login, UI показывает понятную ошибку и не скрывает stderr полностью.

## 8. Ошибки и edge cases

Приложение должно обрабатывать:

- пустой URL;
- URL не из Instagram;
- недоступный пост;
- приватный пост;
- пост, требующий авторизацию/cookies;
- сетевую ошибку;
- ошибку `yt-dlp`;
- ошибку `ffmpeg`;
- неподдержанный media type;
- carousel с несколькими файлами;
- повторный импорт той же ссылки.

Для первого MVP повторный импорт можно разрешить как отдельную запись. Дедупликацию добавим позже.

## 9. Тестирование

Минимальные проверки:

- URL validation принимает `instagram.com/p/...` и `instagram.com/reel/...`;
- URL validation отклоняет пустые и не-Instagram URL;
- parser корректно классифицирует image/video по metadata;
- import command возвращает структурированную ошибку, если `yt-dlp` не найден;
- gallery показывает элементы из `index.json`;
- клик по gallery item меняет preview;
- для видео вызывается extraction first frame;
- ошибка `ffmpeg` не ломает весь импорт, если видео уже скачано.

Интеграционные проверки с реальными Instagram URL выполняются вручную, потому что результат зависит от сети, cookies, доступности поста и текущей поддержки Instagram в `yt-dlp`.

## 10. Не входит в первый MVP

- автопубликация в Instagram;
- официальный Meta API;
- авторизация Instagram;
- cookies manager;
- генерация изображений;
- генерация видео;
- тренд-скоринг;
- caption/hashtag generation;
- cloud sync;
- многопользовательский режим.
