# Самостоятельные установщики macOS и Windows — дизайн

## Цель

Пользователь может передать человеку один файл `instal.command` для macOS или
`instal.bat` для Windows. При запуске в новой папке файл автоматически ставит
требуемые инструменты, разворачивает последний стабильный релиз Instagram
Import Studio в этой же папке, устанавливает зависимости и запускает локальное
приложение в браузере. Полученная копия затем обновляется штатным
`update.command` или `update.bat`.

## Границы и ограничения

- Репозиторий установки: `https://github.com/2lagames/insta_model.git`.
- Стабильный релиз — Git-тег строго формата `vX.Y.Z`; установщик выбирает
  наибольший такой тег по Git version sort.
- Установка происходит в каталоге самого установщика. Поэтому используется
  `git init` + fetch/checkout, а не `git clone <url> .`: файл установщика
  уже делает каталог непустым.
- Установщик не перезаписывает существующую копию приложения: если в каталоге
  уже есть `.git` или `package.json`, он останавливается и предлагает
  воспользоваться `update`.
- macOS получает Homebrew через официальный установщик, если `brew` отсутствует,
  затем ставит `git` и `node`. Homebrew может запросить пароль администратора.
- Windows использует предустановленный `winget` для тихой установки Git и
  Node.js LTS. Если `winget` отсутствует или установка завершается ошибкой,
  скрипт сохраняет окно терминала с конкретной ошибкой.
- Сетевой код получает только публичные установочные пакеты Homebrew, Git,
  Node.js и публичный GitHub-репозиторий; ключи и локальные данные в процесс не
  вовлекаются.

## Компоненты

### `instal.command`

Bash-скрипт с executable-битом. Он переходит в свой каталог, проверяет, что
там нет существующей установки, устанавливает/активирует Homebrew при
необходимости, выполняет `brew install git node`, затем инициализирует Git.
После получения тегов он определяет последнюю версию при помощи `node`,
забирает только этот тег, делает detached checkout, выполняет `npm ci` и
запускает `npm run dev`. Все ошибки остаются видимы до нажатия Enter.

### `instal.bat`

Командный файл Windows выполняет те же шаги: рабочий каталог — `%~dp0`,
предварительно исключается существующая установка, а `winget` ставит
`Git.Git` и `OpenJS.NodeJS.LTS` с параметрами, исключающими интерактивное
подтверждение. После обновления PATH для текущего процесса скрипт инициализирует
репозиторий, получает последний релизный тег, запускает `npm ci` и `npm run dev`.
Любая ошибка вызывает `pause` и ненулевой код выхода.

### Обновляющие скрипты

`update.command` и `update.bat` сохраняют проверку Git, Node.js и npm, но вместо
`git pull --ff-only` получают теги origin, выбирают последний `vX.Y.Z`,
проверяют отсутствие изменённых отслеживаемых файлов, переключаются на тег,
запускают `npm ci` и сервер. Наличие переданного пользователем `instal.*` как
неотслеживаемого файла не является ошибкой. Это гарантирует, что установленная
копия не перейдёт на непомеченный коммит разработки.

## Поток установки и обновления

1. Пользователь создаёт пустую папку и копирует в неё подходящий `instal.*`.
2. Скрипт обеспечивает Git и Node.js LTS для своей ОС.
3. Скрипт инициализирует каталог как Git-репозиторий, подключает `origin`,
   получает теги и разворачивает максимальный `vX.Y.Z`.
4. `npm ci` восстанавливает зависимости из фиксированного lockfile.
5. `npm run dev` открывает интерфейс на `http://localhost:5173`; закрытие
   окна терминала останавливает сервер.
6. Позднее `update.*` повторяет шаги 3–5, но сначала прекращает работу при
   изменениях отслеживаемых файлов, которые могли бы быть затёрты.

## Проверка

- Контрактные Vitest-тесты читают оба установщика, оба обновлятора и README:
  проверяют URL репозитория, автоматическую подготовку prerequisites,
  поиск тегов, `npm ci`, видимый foreground-запуск, безопасную обработку
  существующей установки и executable-бит macOS-файла.
- `bash -n instal.command update.command` проверяет shell-синтаксис.
- Интеграционный прогон в новой временной папке копирует `instal.command`,
  запускает его с уже доступными Git/Node, убеждается, что появился `.git`,
  `package.json`, обновлятор и корректный тег, запускает проект, ожидает
  `/api/health` и останавливает только созданный тестовый процесс.
- В этой среде невозможно нативно исполнить `.bat`; его корректность покрыта
  контрактным тестом. Для Windows используется синтаксис Command Prompt без
  Unix-зависимостей.
